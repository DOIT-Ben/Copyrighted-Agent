# Tests

这个目录不是为了补几条示例测试，而是为了建立后续开发的测试地基。

当前测试分为 4 层：

- `tests/unit/`: 纯逻辑、纯契约、纯文本规则
- `tests/integration/`: ZIP 导入、流水线、API
- `tests/non_functional/`: 安全、性能、稳定性要求
- `tests/helpers/`: 契约辅助函数与通用断言

当前测试骨架遵循一个原则：

- 当目标模块尚未实现时，测试会以 `xfail` 风格的“契约未实现”方式提示
- 一旦目标模块出现，测试会开始真正校验行为

这意味着这些测试可以先落库，再驱动后续实现，而不会要求我们今天就把整个系统写完。

推荐开发顺序：

1. 先让 `tests/unit/` 中的领域模型与纯规则测试转绿
2. 再让 `tests/integration/` 中的 ZIP 主路径转绿
3. 再让 API 与 Web 主路径转绿
4. 最后把真实踩过的坑补成回归测试，例如：
   - 中文 / Windows 非法文件名 ZIP
   - Zip Slip
   - 可执行文件混入
