# 前端卡片密度修正构建日志
## 日期

- 2026-04-21

## 问题

- `/ops` 页面的部分卡片和摘要块被固定列数强行压窄。
- 在中等宽度屏幕下，卡片会出现“又高又瘦”的比例，标题和正文被挤压。
- 主要问题区域：
  - `kpi-grid-ops`
  - `summary-grid`
  - `ops-status-grid`
  - `command-cluster-grid`
  - `closeout-callout`

## 本轮处理

- 把固定列数改成带最小宽度的自适应网格。
- 明确规则：
  - 空间不够时优先换到下一排。
  - 不再为了保持单行而压扁卡片。
- 补充状态卡和 closeout callout 的换行逻辑：
  - 顶部区域允许 `flex-wrap`
  - 中等宽度下主动改为纵向排列
- 调整运维页数值区的对齐逻辑：
  - 不再限制 `strong` 只能占 70%
  - 窄屏时改为左对齐堆叠

## 代码落点

- `app/web/static/styles.css`

## 用户可见结果

- 运维页卡片会更宽。
- 卡片数量多时优先下排，而不是继续横向挤压。
- 标题、状态徽标和说明文案可读性恢复。
