# 207 跨材料差异证据增强日志

## 本轮目标

- 让跨材料一致性问题不再只报“有冲突”，而是直接给出各材料当前识别值。
- 把合作协议纳入软件名称、版本号一致性比对。
- 新增“系统 / 平台 / APP / 小程序”这类口径混用检测。

## 实施内容

- 在 `app/core/reviewers/rules/agreement.py` 中补充协议元数据提取：
  - `software_name`
  - `version`
  - `company_name`
- 在 `app/core/reviewers/rules/cross_material.py` 中增强：
  - 名称一致性输出具体来源值
  - 版本一致性输出具体来源值
  - 日期一致性输出双方日期
  - 排序一致性输出双方顺序
  - 术语口径混用检测
- 在 `tests/unit/test_rule_review_contracts.py` 中新增跨材料差异值与术语混用测试。

## 预期收益

- 报告页里的差异描述会更接近业务审查语言，例如：
  - 信息采集表写什么
  - 说明文档写什么
  - 合作协议写什么
- 用户可以更快定位该改哪份材料，而不是只知道“存在不一致”。
