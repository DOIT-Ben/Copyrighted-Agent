# Development Overview

## Goal

把当前以脚本为主的软著材料审查工具，落成一个可运行的 Web MVP。

本轮目标：

- 支持上传 ZIP
- 支持两种导入模式
- 自动解压、分类、聚合材料
- 执行规则审查
- 生成材料级与项目级报告
- 提供网站界面查看结果
- 以测试驱动方式推进开发
- 把开发过程、问题、修复和经验沉淀到 `docs/dev/`


## MVP Scope

本轮只做 MVP，不做复杂权限、队列、数据库持久化和真实 AI 供应商接入。

MVP 包含：

- FastAPI Web 应用
- Jinja2 模板页面
- 基于文件系统和内存的 Submission / Case / Material 运行态存储
- ZIP 模式 A / B 基础支持
- 材料分类
- 解析统一入口
- 规则审查
- Mock AI 结果
- 报告渲染
- 测试执行与问题修复


## Design Direction

参考本地 `ui-ux-pro-max-skill` 的设计原则，采用以下 UI 方向：

- 以信息架构清晰为首要目标
- 以高对比、低噪音、强层次的专业审查台为视觉基调
- 使用设计令牌统一颜色、阴影、圆角、间距、排版
- 响应式布局优先支持桌面，再兼顾移动端
- 保留足够白空间，减少“管理后台模板感”


## Delivery Philosophy

开发顺序：

1. 先落文档与 todo
2. 先做核心领域与服务
3. 再做 API 和 Web 页面
4. 再跑测试
5. 根据测试结果修复
6. 最后补经验手册
