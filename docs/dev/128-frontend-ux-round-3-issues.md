# 前端体验优化第三轮问题记录
## 日期

- 2026-04-20

## 发现的问题

### 1. 首页导入模式说明不够前置

- 现象：
  - 用户先看到上传表单，再理解模式差异，容易在模式 A 和模式 B 之间误选。
- 风险：
  - 后续批次整理成本增加，人工需要反复重组材料。
- 处理：
  - 在上传表单前增加模式对比卡片，并突出推荐起点。

### 2. 批次详情页操作台认知负担过高

- 现象：
  - 更正类型、指派项目、创建项目、合并项目、重跑审查全部平铺在同一层。
- 风险：
  - 操作人员容易跳步骤，先重跑再纠偏，导致返工。
- 处理：
  - 拆成“材料纠偏 / 项目编排 / 复核重跑”三组，并在页面上方增加顺序提醒。

### 3. 运维页容易看见信息但不容易判断顺序

- 现象：
  - 运维页指标多、面板多，但缺少明确的观察路径。
- 风险：
  - 用户可能直接看趋势或探针，却没有先判断发布闸门是否允许继续。
- 处理：
  - 增加顺序面板和页内跳转，把联调判断流程固定下来。

### 4. 移动端表格过度依赖横向滚动

- 现象：
  - 多列表格在小屏设备上阅读成本高，信息对照容易丢失。
- 风险：
  - 运营和复核人员在移动端快速核对数据时效率明显下降。
- 处理：
  - 共享表格改为在小屏下输出堆叠卡片式单元格，保留列名标签。

### 5. 共享导航缺少当前页和跳转主内容能力

- 现象：
  - 左侧导航只有视觉高亮，没有补足语义态；键盘用户缺少快速跳过侧栏的路径。
- 风险：
  - 无障碍体验不完整，长页工作流会增加键盘导航成本。
- 处理：
  - 增加 `aria-current="page"`、`skip-link` 和主内容锚点。

## 残余风险

- 本轮主要依赖模板和样式层改造，真实视觉效果仍需要浏览器实际渲染持续观察。
- 折叠式操作区虽然降低了初始密度，但如果后续再继续增加动作，需要补二次分层规则。
- 移动端堆叠表格适合阅读，不适合极高频的跨列对比；如果后续出现更重的移动使用场景，可能需要专门的移动摘要卡片。
