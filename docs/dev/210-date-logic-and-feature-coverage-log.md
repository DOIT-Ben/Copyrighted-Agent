# 210 日期逻辑与功能呼应增强日志

## 本轮目标

- 继续细化合作协议日期逻辑。
- 让“说明文档写了什么功能，源码有没有呼应”变成可执行检查。
- 同步优化结果页的业务化提示文案。

## 本轮实现

- `app/core/reviewers/rules/agreement.py`
  - 新增带标签日期识别：
    - 签订日期
    - 开发完成日期
    - 申请日期
  - 支持检查：
    - 签署时间是否晚于开发完成时间
    - 签署时间是否晚于/等于申请时间
    - 签署时间距离申请时间是否过近
- `app/core/reviewers/rules/document.py`
  - 抽取说明文档中的功能词，供跨材料功能呼应使用
- `app/core/reviewers/rules/source_code.py`
  - 抽取源码中的函数名/类名/关键英文信号，供跨材料功能呼应使用
- `app/core/reviewers/rules/cross_material.py`
  - 新增文档功能词与源码信号的呼应检查
- `app/web/page_report.py`
  - 为新增和近期补的规则增加更友好的结果摘要

## 预期收益

- 报告结果会更接近人工审查口吻：
  - 协议时间是不是太靠近申请时间
  - 文档里写了很多功能，但源码里是否没有对应支撑
