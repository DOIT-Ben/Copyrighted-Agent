# 前端体验优化第三轮构建日志
## 日期

- 2026-04-20

## 本轮范围

- 首页：强化导入模式判断、提交前检查和起步顺序。
- 批次详情页：强化人工复核顺序，重组操作台。
- 运维页：强化联调与发布判断顺序。
- 共享层：补无障碍增强与移动端表格展示能力。
- 样式层：补新组件、响应式规则和交互反馈。

## 实际改动

- `app/web/view_helpers.py`
  - 导航链接增加 `aria-current="page"`。
  - 页面布局增加 `skip-link` 和主内容锚点。
  - 表格渲染增加 `_table_cell(...)`，为移动端堆叠展示输出标签和值结构。
  - `panel(...)` 头部文案区增加 `panel-head-copy`，方便标题和描述稳定排版。
- `app/web/page_home.py`
  - 增加导入模式对比卡片。
  - 增加“提交前检查”提示。
  - 增加“开始前看这里”顺序面板。
  - 调整首页主面板宽度占比，让导入台成为更明确的主操作区。
- `app/web/page_submission.py`
  - 增加“这个批次先怎么处理”顺序面板。
  - 将人工操作台拆为“材料纠偏 / 项目编排 / 复核重跑”三个折叠组。
  - 保持原有动作和后端接口不变，只重构认知顺序和排布。
- `app/web/page_ops.py`
  - 增加“这页应该怎么看”顺序面板。
  - 提供页内快捷跳转，减少运维用户来回找区块。
- `app/web/static/styles.css`
  - 扩展工作区、面板、按钮、卡片、表格和侧栏的视觉细节。
  - 新增 `compare-grid`、`sequence-board`、`operator-group-grid` 等组件样式。
  - 新增移动端堆叠表格规则。
  - 保持可见焦点和 `prefers-reduced-motion` 支持。

## 关键设计决定

- 没有把管理台做成营销页，而是继续沿用数据密集型工作台方向。
- 优先解决“先做什么、后做什么”，而不是继续堆更多强调色或装饰元素。
- 移动端优化优先落在表格阅读上，因为这类页面的主要成本来自信息扫描。
- 人工操作区改成折叠分组，是为了把“纠偏、编排、重跑”这三个阶段拉开，而不是改变动作本身。

## 本轮产出

- 用户进入首页后，能更快判断该选模式 A 还是模式 B。
- 用户打开批次详情后，能先看待复核，再看材料矩阵，再做人工干预。
- 用户打开运维页后，能先判断是否可以继续发布验证，再看通道与趋势。
- 用户在小屏设备上查看表格时，不再依赖横向滚动作为主要阅读方式。
