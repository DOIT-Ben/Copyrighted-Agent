# 211 报告来源分组与在线填报维度骨架日志

## 本轮目标

- 让结果页可以按材料来源查看问题，而不是只看统一的问题清单。
- 把“在线填报信息审查”补成正式维度，先支持规则编辑、提示词和结果占位。

## 本轮实现

- `app/web/page_report.py`
  - 新增“按材料来源看问题”面板
  - 支持将问题按以下来源分组展示：
    - 信息采集表
    - 说明文档
    - 源代码
    - 协议
    - 跨材料比对
    - 在线填报
- `app/core/services/review_rulebook.py`
  - 新增 `online_filing` 维度规则壳
  - 当前包含：
    - 软件分类选择正确
    - 开发方式选择正确
    - 主体类型选择正确
    - 地址信息足够精确
    - 填报日期与材料口径一致
- `app/core/services/review_profile.py`
  - 将 `online_filing` 纳入维度目录，支持前端查看和编辑规则
- `app/core/services/review_dimensions.py`
  - 为 `online_filing` 增加结果占位
  - 当前会明确提示“尚未接入独立在线填报数据”

## 当前边界

- 仓库里还没有独立的在线填报材料类型，也没有系统导出字段持久化入口。
- 所以这轮先补的是：
  - 维度骨架
  - 规则编辑能力
  - 提示词承载能力
  - 结果页占位能力

## 下一步衔接

- 接入在线填报截图、导出文件或结构化字段后，可以直接把这些规则接到实际执行逻辑上，无需重做前端和配置骨架。
