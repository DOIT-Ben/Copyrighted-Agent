# 首页顶部布局重排构建日志

## 日期

- 2026-04-21

## 背景

- 首页顶部的 `import-console` 与并列卡片同时占据一行。
- 在桌面较窄宽度下，两个重组件互相抢宽度，导致卡片过瘦、文字被挤压、表单区域不自然拉长。
- 这类问题不是单个尺寸参数可以修好，根因是顶部信息架构错误。

## 本轮目标

- 彻底取消首页顶部“双重组件并排抢宽度”的结构。
- 让空间不足时优先下排，而不是继续横向压缩。
- 保持现有后台工作台视觉系统，不改成营销页。

## 实施内容

- 重写 `app/web/page_home.py` 首页顶部结构：
  - 取消 `lead-grid`
  - 将 `导入台` 调整为 `dashboard-grid` 内的 `span-12`
  - 将 `可信信号` 调整为独立的 `span-12` 区块并放在其下方
- 重做 `导入台` 内部布局：
  - 新增 `import-console-grid`
  - 左侧为模式说明、摘要卡、模式对比卡
  - 右侧为单列导入表单与导入前/提交前说明
- 重做 `可信信号` 内部布局：
  - 由竖向窄堆叠改为 `trust-signal-grid`
  - 拆成三个可自适应宽度的信号卡片
- 清理旧样式残留：
  - 移除 `lead-grid`
  - 移除 `panel-import-lead`
  - 移除 `panel-trust-rail`
- 新增响应式策略：
  - 在 `1400px` 以下，`import-console-grid` 提前切为单列
  - 在 `960px` 以下，摘要卡与可信信号卡统一转单列

## 结果

- 首页顶部主入口不再因为并排关系被压瘦。
- 表单区恢复为单列操作区，不再和文案块互相挤压。
- 宽度不足时整体下排，符合“空间不够就往下排”的要求。
