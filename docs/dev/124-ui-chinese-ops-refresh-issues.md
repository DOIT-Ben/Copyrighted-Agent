# 中文运维台重构问题记录

## 日期

- 2026-04-20

## 发现的问题

### 1. 页面可见文案长期中英混杂

- 现象：
  - 首页、批次页、项目页、报告页和运维页混有英文标题、按钮和说明。
- 风险：
  - 用户切页面时体验断层，像多个半成品拼起来。
- 处理：
  - 本轮统一替换为中文管理台语义，并保留必要的技术命令原文。

### 2. 运维页信息层级过平

- 现象：
  - `/ops` 的命令区、趋势区和探针区都在同层级堆叠，扫读成本高。
- 风险：
  - 用户很难快速判断先做什么，再做什么。
- 处理：
  - 把命令区按阶段分组。
  - 给趋势区增加摘要和结论提示。

### 3. 历史改动里存在乱码残留风险

- 现象：
  - 部分旧测试和旧文案仍使用乱码字符串或英文旧断言。
- 风险：
  - 页面已修好，但测试仍会误报失败。
- 处理：
  - 统一更新 UI 契约测试。
  - 回归时额外跑 `test_web_source_contracts.py`，确认没有新的 mojibake 标记。

## 本轮经验

- 中文管理台不能只“翻译标题”，必须连带调整：
  - 卡片密度
  - 表格标题
  - 按钮文案长度
  - 页面分区顺序
- 对运维页来说，命令分组比继续加更多按钮更重要。
- 改 UI 文案时，必须同步改测试契约，否则后续回归结果没有参考价值。
