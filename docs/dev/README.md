# Dev Notes Index

`docs/dev/` contains historical build logs, validation notes, issue notes, and playbooks.

The folder is intentionally append-only for traceability. Use these categories when adding new notes:

- `*-plan.md`: planned scope and acceptance criteria.
- `*-build-log.md` or `*-log.md`: implementation summary.
- `*-validation.md`: commands, results, and test evidence.
- `*-issues.md`: risks, defects, and follow-up items.
- `history/`: generated historical snapshots and archived run artifacts.
