# 审查结果页证据链展示开发日志

日期：2026-04-25

## 背景

用户反馈“审查结果”页面缺少直观的信息链，无法直接看出：

- 发现了什么不足
- 系统怎么查到的
- 命中了哪条审查规则
- 为什么会给出当前结论

## 本次调整

已在 `app/web/page_report.py` 的项目报告页补充三块核心结果视图：

- `发现了什么不足`
  - 用表格串联问题级别、问题名称、命中维度、命中规则、发现依据、建议动作
- `怎么判定出来的`
  - 按维度展示摘要、发现证据、规则目标、LLM 关注点
- `用了哪些审查规则`
  - 直接展示本次报告实际启用的规则目标和模型关注点

同时更新了页面内导航锚点，支持从页内导航直接跳到这三个版块。

后续又补充了一层更面向业务用户的诊断区：

- `先改这些地方`
  - 将原始规则命中翻译成更直白的人话结果
  - 优先展示“名称不一致”“版本号不一致”“合作协议错别字”“协议日期不一致”“说明文档术语不一致”“源码存在乱码”等可直接执行的修改项
  - 目标是让用户先看到“哪里不对”，再决定是否展开证据链查看依据

## 回归范围

- 更新 `tests/integration/test_web_mvp_contracts.py`
- 新增报告页关键文案断言：
  - `先改这些地方`
  - `发现了什么不足`
  - `怎么判定出来的`
  - `用了哪些审查规则`

## 说明

本次改动优先解决“结果可解释性”和“证据可追溯性”，未改动后端审查规则生成逻辑与 LLM 提供方接入逻辑。
