# 响应式密度优化构建日志

## 日期

- 2026-04-21

## 目标

- 处理首页之外的中间宽度区间挤压问题。
- 优化批次详情、项目详情、报告页的栅格分配。
- 让表格更早进入卡片式阅读，而不是依赖横向滚动。

## 本轮改动

- 调整全局断点：
  - 侧边栏从 `1200px` 提前到 `1320px` 收起
  - 新增 `1180px` 断点，`span-4/5/6/7/8` 提前转为整行
  - 表格卡片化从 `720px` 提前到 `960px`
- 调整通用样式：
  - 新增 `span-6`
  - `dossier-list` 改为 `auto-fit + minmax`，不再固定双列
  - 报告正文容器限制为更适合阅读的最大行宽
- 重排批次详情：
  - `导入摘要` 改为 `span-5`
  - `待复核队列` 改为 `span-7`
  - `导出中心` 独占整行
  - `产物浏览` 独占整行
  - `人工干预台` 独占整行
- 重排项目详情：
  - `风险队列` / `材料矩阵` 调整为 `8/4` 组合
  - `AI 辅助研判` / `项目信号` 改为较窄侧栏
- 重排报告页：
  - `报告阅读器` 独占整行
  - `报告上下文` 下沉到正文下方，避免与正文抢宽度

## 涉及文件

- `app/web/static/styles.css`
- `app/web/page_submission.py`
- `app/web/page_case.py`
- `app/web/page_report.py`

## 预期效果

- 详情页在 13 寸笔记本和中等窗口宽度下不再出现明显横向挤压。
- 复杂表格更早转成分块阅读，减少横向滚动。
- 报告页从“分栏信息页”更接近“阅读器页面”。
