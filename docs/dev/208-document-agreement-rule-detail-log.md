# 208 文档与协议细项规则落地日志

## 本轮目标

- 把说明文档和合作协议中较粗的规则，继续拆成更接近业务模板的可执行细项。
- 让结论能直接告诉用户到底缺了哪一项，而不是只提示“有问题”。

## 本轮实现

- `app/core/reviewers/rules/document.py`
  - 补充运行环境细项检查
    - 硬件要求
    - 软件环境要求
  - 补充术语口径混用检查
    - 系统
    - 平台
    - APP
    - 小程序
  - 补充页眉信号检查
    - 是否存在“软件全称 + 版本号”的稳定页眉信号
- `app/core/reviewers/rules/agreement.py`
  - 补充关键人员缺失检查
  - 补充审批手续检查
    - 科研合同审批表
    - 技术开发合同勾选/表述

## 预期收益

- 用户在结果页里能更直观地看到：
  - 运行环境缺的是硬件要求还是软件环境要求
  - 协议缺的是审批表还是合同类型表述
  - 文档是不是存在“系统/平台”混用
